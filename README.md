# DemoExam

Desktop-приложение на Avalonia UI для работы с товарами и заказами магазина
«ВелосипедДрайв». Приложение подключается к PostgreSQL, показывает каталог
товаров, поддерживает авторизацию пользователей и CRUD-операции для товаров и
заказов в зависимости от роли.

## Возможности

- вход по логину и паролю или вход в режиме гостя;
- просмотр каталога товаров с фотографиями, ценами, скидками и остатками;
- поиск товаров по названию, производителю и поставщику;
- сортировка товаров по цене;
- фильтрация товаров по диапазону скидки;
- добавление, редактирование и удаление товаров для администратора;
- просмотр, поиск, добавление, редактирование и удаление заказов;
- разграничение доступных действий по ролям пользователя.

## Стек

- .NET 8
- Avalonia UI 12
- Avalonia.Controls.DataGrid
- PostgreSQL
- Npgsql

## Требования

- .NET SDK 8.0 или новее
- PostgreSQL 16 или совместимая версия
- база данных `demoexam`

По умолчанию приложение использует строку подключения:

```text
Host=localhost;Port=5432;Username=postgres;Password=0000;Database=demoexam
```

Если параметры PostgreSQL отличаются, измените `connectionString` в файлах:

- `Views/Login/LoginWindow.axaml.cs`
- `Views/Products/ProductWindow.axaml.cs`
- `Views/Products/ProductEditWindow.axaml.cs`
- `Views/Orders/OrderWindow.axaml.cs`
- `Views/Orders/OrderEditWindow.axaml.cs`

## Подготовка базы данных

В корне проекта есть дамп `demoexam.sql`. Для локального запуска создайте базу и
импортируйте дамп:

```bash
createdb -U postgres demoexam
psql -U postgres -d demoexam -f demoexam.sql
```

Дамп содержит таблицы:

- `categories`
- `manufacturers`
- `order_items`
- `order_statuses`
- `orders`
- `pickup_points`
- `products`
- `roles`
- `suppliers`
- `users`

Также используются представления:

- `view_products`
- `view_orders`

## Запуск

В корне проекта выполните:

```bash
dotnet restore
dotnet build
dotnet run
```

После запуска откроется окно авторизации. Для гостевого просмотра можно нажать
кнопку входа гостя.

## Структура проекта

```text
.
├── App.axaml                # ресурсы и стили приложения
├── App.axaml.cs             # запуск главного окна
├── Program.cs               # точка входа Avalonia

├── Models/                  # модели данных

    ├── Products/            # каталог и редактирование товаров
    └── Orders/              # список и редактирование заказов
```

## Роли

Доступность действий определяется названием роли пользователя:

- `Гость` - просмотр каталога без поиска, фильтров и управления заказами;
- `Авторизированный клиент` - просмотр каталога без административных действий;
- `Менеджер` - просмотр каталога с поиском, сортировкой и фильтрами;
- `Администратор` - полный доступ к управлению товарами и заказами.

## Примечания

- Фотографии товаров загружаются из папки `Assets` по имени файла, указанному в
  поле `photo` таблицы `products`.
- Если фотография не указана или файл не найден, используется
  `Assets/picture.png`.
- Пароли пользователей сейчас проверяются напрямую по значению из таблицы
  `users`; хеширование паролей в проекте не реализовано.
